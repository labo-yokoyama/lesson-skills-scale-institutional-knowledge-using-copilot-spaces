# OctoAcme プロジェクト管理プロセス概要

このドキュメントは、OctoAcmeで用いられる標準的なプロジェクト管理プロセスの要点をまとめています。全メンバーに一貫性のある作業体系を提供し、オンボーディングや継続的な改善を支援します。

---

## ワークフローと主要フェーズ

OctoAcme のプロジェクトは、「イニシエーション（構想・立ち上げ）」「計画」「実行／進捗管理」「リリース・展開」「振り返りと継続的改善」の5段階を踏んで運営されます。各フェーズでは、プロジェクトワンペーパーやバックログ、リスク登録簿といった成果物の作成・更新を通じて、進捗や責任・リスクを可視化します。

## 役割（ペルソナ）

- **プロジェクトマネージャー（PM）**: スケジュール調整、リスク管理、ドキュメンテーションを主導。
- **プロダクトマネージャー（PdM）**: ビジョン定義、バックログ優先付け、成果指標の管理。
- **開発者**: 実装・設計・テスト、リファクタリング／レビューを担当。
- **QA担当/テスター**: テスト計画と品質保証をリード。
- **ステークホルダー**: フィードバック・承認・意思決定に関与。

## コミュニケーション戦略

週次のPM＋PdM同期、デリバリーチームのスタンドアップ、月次ステークホルダー向け進捗報告など、役割ごとに体系化されたコミュニケーションが実施されます。リスクや課題は、定例会でエスカレーションパスを明確にしつつ共有されます。

## 品質保証と継続的改善

- Pull Requestごとの自動テスト・レビュー必須化、CI/CDによる品質担保
- ユニットテスト／統合テスト、重要フローのスモークテスト
- リリース前後のリスクチェックとインシデント対応フロー
- 各サイクルの振り返りとアクションアイテム化による改善
